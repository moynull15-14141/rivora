# 14 — Coding Rules

- Variable/function naming: **camelCase**
- Component/Type naming: **PascalCase**
- TypeScript-এ `any` ব্যবহার করা যাবে না
- **ESLint** চালু রাখতে হবে (commit-এর আগে lint check)
- **Prettier** দিয়ে format consistent রাখতে হবে
- প্রতিটি API response-এর জন্য consistent shape (`{ success, data, error }`)
- Component ছোট রাখা, একটার বেশি responsibility না দেওয়া
